# 快乐小狗项目总结

更新时间：2026-05-31  
下一次继续：2026-06-01

## 项目定位

快乐小狗是一个 macOS 桌面陪伴型效率小工具。它不是传统的大窗口应用，而是一个常驻桌面的小狗助手，通过轻量提醒、情绪反馈、番茄钟和菜单栏入口，帮用户在工作时保持节奏。

当前产品方向：

- 第一优先级是“可爱、轻、不打扰”，小狗本身应该是主视觉。
- 效率功能要藏在顺手的位置，不要一上来占满界面。
- 操作应该稳定，点击按钮不能导致小狗位置跳动。
- 小狗可以被拖动到桌面合适位置，并记住位置。
- 下一阶段功能重点转向真实工作高频能力，剪贴板历史已经完成第一版，后续继续做搜索、收藏、固定条目这类增强。

## 已完成：剪贴板历史

用户明确想要的方向是“工作中常用的复制粘贴增强”：复制过的内容可以记录 20 条左右，之后能拉出来找回刚才复制过的东西。

已完成 MVP：

- 自动记录最近复制的文本，默认保留 20 条。
- 自动记录最近复制或截图产生的图片，和文本混排展示。
- 自动记录最近复制的文件路径，和文本、图片混排展示。
- 连续复制同一段内容时去重，避免重复刷屏。
- 数据只存在本机，不上传。
- 菜单栏新增“剪贴板历史”入口。
- 独立窗口能展示最近复制内容。
- 历史窗口支持按文本、文件名、类型或图片尺寸搜索。
- 支持置顶常用剪贴板记录，置顶项优先展示并优先保留。
- 点选某一条后，把该内容重新写回系统剪贴板，方便继续粘贴。
- 支持一键清空剪贴板历史。
- 支持单条删除。
- 支持暂停或恢复剪贴板文本记录。
- 有基础敏感文本过滤，疑似密码、token、secret 的短文本不会记录。

技术实现：

- `ClipboardCoordinator.swift` 用 `NSPasteboard.general.changeCount` 轮询监听剪贴板变化。
- `WorkdogStore` 增加剪贴板历史数组、最大条数、启停开关和清空方法。
- 使用 `UserDefaults` 存 JSON 编码的历史元数据。
- 图片文件存到 `~/Library/Application Support/HappyWorkdog/ClipboardImages`，避免把大图塞进偏好文件。
- `ClipboardHistoryView` 提供独立历史窗口。
- 点击历史项通过 `NSPasteboard` 写回系统剪贴板。

后续增强方向：

1. 增加常用片段收藏。
2. 提供更可解释的敏感内容过滤设置。
3. 支持富文本历史。

## 当前功能

### 桌面小狗

- 透明无边框桌面窗口，常驻在桌面上方。
- 小狗窗口默认显示在屏幕右下附近。
- 支持拖动小狗窗口，并把窗口位置保存到 `UserDefaults`。
- 支持点击小狗主体触发“摸摸”反馈，并累计今日摸摸次数。
- 小狗有轻微上下浮动动画。
- 小狗外观支持：
  - 名字：默认“快乐小狗”
  - 体型：圆润、蓬松、利落、紧凑
  - 毛色：金黄、奶油、可可、灰黑、天蓝、薄荷
  - 缩放：通过设置调整小狗大小
- 小狗状态支持：
  - idle：平常
  - happy：开心
  - energized：打气
  - proud：完成专注后的骄傲状态

### 对话气泡

- 小狗上方有紧凑气泡，展示当前提醒或状态文案。
- 气泡颜色会跟随提醒类型变化。
- 当前已经从一开始偏大的对话框收敛成更小、更贴近桌宠的样式。
- 明天还可以继续打磨箭头形态、阴影、位置和文字密度。

### 快捷按钮

小狗下方有一排小按钮：

- 番茄钟入口
- 区域截图到剪贴板
- 喝水记录
- 休息记录
- 打气记录

交互原则：

- 点击喝水、休息、打气按钮会记录一次今天的行为，不发送系统通知。
- 自动触发的喝水、休息、打气才累计为提醒次数。
- 点击按钮只触发功能，不移动小狗。
- 拖动小狗和点击按钮要走清晰分离的事件路径。
- 不再使用动态 `window.ignoresMouseEvents`，这类方案之前导致按钮失效。

### 截图方案

当前截图已经从“打开系统截图工具”演进为小狗自己的微信式截图覆盖层，主要代码在 `Sources/happy-workdog/ScreenshotCoordinator.swift`。

触发入口：

- 小狗底部相机按钮。
- 菜单栏“区域截图到剪贴板”。
- 两个入口都走 `WorkdogAppDelegate.takeScreenshot(_:)`。

当前交互：

- 触发截图时先临时隐藏小狗，避免小狗被截进去。
- 使用 `CGDisplayCreateImage` 抓取当前屏幕快照。
- 每个屏幕创建一个 `ScreenshotSelectionWindow` 全屏覆盖层。
- 桌面会被“定住”，覆盖一层暗色遮罩。
- 第一次拖动创建截图选区。
- 松手后选区不会立即完成，可以继续移动整体选区或拉 8 个控制点调整范围。
- 点击蓝色确认按钮或按回车，才会裁剪并写入剪贴板。
- 点击红色取消按钮或按 ESC 取消。

当前标注能力：

- 箭头。
- 矩形框。
- 画笔。
- 文案文字。
- `⌘Z` 可以撤销最近一次标注。
- 文字输入框是透明无边框样式，只显示红色文字。
- 文字工具可以连续写多段：回车提交后仍保持文字模式，也可以直接点新位置继续写。
- 标注可移动：切回普通模式后，拖动标注附近即可移动它。
- 标注移动时不显示虚线框，避免画面杂乱。
- 标注只能在截图选区内移动，不会拖出最终图片。
- 确认截图时，标注会按 Retina 原始像素级别合成到裁剪图片里。

权限和签名注意事项：

- 自定义截图仍需要 macOS“屏幕录制 / 屏幕与系统音频录制”权限。
- 不再使用自己写的权限提示弹窗，只依赖系统弹窗和小狗气泡文案。
- 之前遇到过授权后仍不可用的问题，根因是打包脚本每次 ad-hoc 签名导致 TCC 绑定的 `cdhash` 变化。
- `scripts/package_app.sh` 已改成稳定 designated requirement：`designated => identifier "com.local.happy-workdog"`。
- 如果以后再次遇到系统设置里看似已授权但截图失败，优先检查 TCC 日志是否有 `Failed to match existing code requirement`。
- 必要时执行：`tccutil reset ScreenCapture com.local.happy-workdog`，然后重新打开 app 并重新授权。

后续可优化：

- 标注颜色和线宽选择。
- 删除单个已选标注。
- 文字编辑已落下的文案。
- 箭头、矩形、文字的大小调整。
- 多屏跨屏拖选目前不是目标，当前按单屏覆盖层处理。

### 菜单栏

菜单栏显示一个爪印图标，提供这些入口：

- 显示或隐藏小狗
- 打开设置
- 暂停或恢复提醒
- 手动记录喝水、休息、打气
- 区域截图到剪贴板
- 查看番茄钟当前时间
- 开始或暂停番茄钟
- 重置番茄钟
- 退出应用

### 提醒系统

当前支持三类提醒：

- 喝水
- 休息
- 打气

提醒能力：

- 可以在设置页分别开启或关闭。
- 可以配置不同提醒间隔。
- 自动提醒会发送 macOS 通知，手动点击会记录行为。
- 系统会把提醒写入日志。
- 提醒触发时，小狗会切换表情和文案。
- 如果开启“提醒时显示小狗”，提醒会把小狗窗口唤起。

日志路径：

```text
~/Library/Logs/HappyWorkdog/reminders.log
```

### 番茄钟

当前番茄钟不是默认展开，而是作为一个功能入口被唤醒。

入口形态：

- 空闲时：小狗下方显示一个小的番茄钟按钮。
- 运行时：入口变成小型时钟状态，显示进度感。
- 点击入口后：弹出紧凑的时钟面板。

番茄钟面板能力：

- 专注、短休、长休三种模式。
- 默认时长：
  - 专注：25 分钟
  - 短休：5 分钟
  - 长休：15 分钟
- 支持开始、暂停、重置、跳到下一段。
- 支持完成专注次数统计。
- 支持每完成若干个专注后进入长休，默认每 4 个专注进入一次长休。
- 专注结束后自动进入休息段。
- 休息结束后自动回到专注段。
- 完成时会触发通知、日志和小狗状态变化。
- 完成专注会累计到今日专注次数。

视觉方向：

- 番茄钟应该更像一个真实时钟，而不是普通卡片。
- 当前已经有圆形表盘、刻度、进度环和指针。
- 明天可以继续增强“时钟感”，例如增加更稳定的刻度层级、指针运动细节、完成时反馈。

### 今日摘要

设置页新增“今日”入口，默认打开今日摘要。

当前统计：

- 喝水记录
- 休息记录
- 打气记录
- 今日完成专注
- 收到自动提醒总数
- 摸摸小狗次数

统计口径：

- 自动提醒只累计提醒次数。
- 底部按钮和菜单栏主动点击累计为行为记录。
- 每天按本地日期自动切换并清零。
- 可以在今日摘要中手动清空今天记录。

## 技术架构

### 基础栈

- 语言：Swift
- UI：SwiftUI
- macOS 桌面能力：AppKit
- 包管理：Swift Package Manager
- 最低系统目标：macOS 13
- 应用形态：SwiftPM executable，打包成 `.app`

### 启动方式

入口文件：

```text
Sources/happy-workdog/main.swift
```

应用通过 `NSApplication` 启动，使用 `WorkdogAppDelegate` 管理窗口、菜单栏和提醒调度。

### 核心模块

```text
Sources/happy-workdog/
  main.swift
  WorkdogWindows.swift
  WorkdogViews.swift
  WorkdogStore.swift
  WorkdogModels.swift
  ReminderCoordinator.swift
```

模块职责：

- `WorkdogWindows.swift`：AppKit 层，负责菜单栏、宠物窗口、设置窗口、窗口拖动和位置保存。
- `WorkdogViews.swift`：SwiftUI 层，负责小狗、气泡、快捷按钮、番茄钟面板、设置页等视觉和交互。
- `WorkdogStore.swift`：应用状态中心，负责提醒设置、小狗外观、番茄钟状态、当前文案和持久化。
- `WorkdogModels.swift`：枚举和模型定义，包括提醒类型、小狗状态、毛色、体型、番茄钟模式、文案。
- `ReminderCoordinator.swift`：提醒调度、macOS 通知、日志写入。

## 桌面窗口与交互方案

当前宠物窗口是 AppKit 的透明无边框 `NSWindow`，内部嵌入 SwiftUI 的 `PetRootView`。

窗口关键配置：

- `styleMask: [.borderless]`
- `backgroundColor = .clear`
- `isOpaque = false`
- `level = .statusBar`
- `collectionBehavior = [.canJoinAllSpaces, .fullScreenAuxiliary, .stationary]`
- `isMovableByWindowBackground = false`
- `ignoresMouseEvents = false`

当前点击和拖动方案：

- SwiftUI 按钮继续使用原生 `Button`。
- `WorkdogWindow.sendEvent(_:)` 只在指定的小狗区域里处理拖拽。
- 番茄钟面板展示时，不开启宠物区域拖拽，避免影响面板按钮。
- 窗口移动后保存位置。

这里是明天最需要谨慎的地方：

- 透明无边框窗口的点击、拖动、穿透很容易互相影响。
- 之前动态切换 `window.ignoresMouseEvents` 会导致按钮点不了，不能再走这个方向。
- 更稳的后续方案是增加一个明确的拖动区域或拖动把手，让用户知道哪里可以拖，也让按钮区域完全不参与拖动判断。

## 当前已知问题与风险

### 1. 点击和拖动仍是高风险区

虽然当前已经比之前稳定很多，但这块仍然是最敏感的技术点。原因是应用同时满足：

- 窗口透明
- 无边框
- SwiftUI 按钮
- AppKit 自定义拖动
- 常驻桌面层级

明天不要大范围改这块。更稳的路线是小步验证：

1. 先确定按钮全部可点。
2. 再确定小狗主体可拖。
3. 最后增加明确拖动把手。

### 2. 视觉还可以继续变可爱

已经做了爪子、毛色、体型、立体阴影和表情，但小狗还有继续提升空间：

- 脸部比例可以更软。
- 耳朵和爪子可以更有层次。
- 眼睛可以更有神。
- 身体阴影和高光可以更自然。
- 完成番茄钟时可以有更明显的开心反馈。

### 3. 对话气泡还需要精修

当前气泡已经变小，但还可以继续优化：

- 箭头形状
- 气泡和小狗头部的相对位置
- 文字换行
- 阴影轻重
- 气泡颜色和不同提醒状态的关系

### 4. 缺少 UI 自动化测试

现在已有单元测试覆盖模型和文案的基本正确性，但还没有自动化验证：

- 按钮是否能点击
- 窗口是否能拖动
- 番茄钟面板是否能打开和关闭
- 设置项是否正确持久化

macOS 桌面 UI 自动化会比普通 Web 麻烦，后面可以先用手动验收清单补上。

### 5. 测试警告

运行测试时会出现已知 warning：

```text
building for macOS-13.0, but linking with dylib '@rpath/Testing.framework/Versions/A/Testing' which was built for newer version 14.0
```

目前测试仍能通过。后续如果要发正式版本，可以再评估是否调整测试框架或最低系统版本。

## 常用命令

构建：

```bash
swift build --disable-sandbox
```

测试：

```bash
swift test --disable-sandbox
```

打包 `.app`：

```bash
./scripts/package_app.sh
```

关闭当前运行的小狗：

```bash
killall happy-workdog
```

打开打包后的应用：

```bash
open -n 'build/Happy Workdog.app'
```

## 明天建议的工作方式

明天不要先大改结构，先做一轮稳定性验收：

1. 启动应用，确认小狗出现。
2. 逐个点击所有按钮，确认都能触发。
3. 拖动小狗，确认窗口跟手。
4. 再点按钮，确认小狗不会跳位置。
5. 打开番茄钟面板，确认面板内按钮都能点。
6. 重启应用，确认小狗位置、外观、设置都保存。

验收通过后，再进入视觉打磨。这样可以避免再次陷入“能拖不能点、能点不能拖”的循环。
